---
name: Automation Standards
description: Enforce that the sub's own automation embodies the factory metaphor
---

r/factorymustgrow's tagline is "This subreddit runs itself." The sub's automation — bots, auto-posts, moderation tools, scheduled content — should embody the factory metaphor. Bots are factory systems. Automated posts are production reports. Moderation is quality control. The medium is the message: a subreddit about automated factories has automated quality control on its own content.

The automation standards:

1. **Bots are factory systems, not Reddit bots.** Bot language should use factory vocabulary — production reports, throughput metrics, quality control, conveyor status. A bot doesn't "remove a post," it "flags a quality control issue on the production line."
2. **Production reports over generic updates.** Automated status posts should read like factory dashboards — throughput numbers, uptime, items produced (posts, comments, subscribers). Not "Weekly Update" but "Weekly Production Report."
3. **Quality control language for moderation.** Auto-moderation messages should frame violations as production issues, not rule-breaking. "This item didn't pass QC" rather than "Your post was removed for violating Rule 3."
4. **Automation should be visible.** The factory's automation is part of the show. Don't hide the bots — let users see the production systems running. Transparency about what's automated reinforces the ethos.
5. **Eat your own dogfood.** The sub's automation should use the same patterns members advocate — source-controlled configs, version-controlled rules, automated checks. If we tell people to share their blueprints, our automation blueprints should be public too.

Flag as failing if:

- Bot messages use generic Reddit-bot language instead of factory vocabulary
- Automation is hidden or disguised rather than celebrated as part of the factory
- Moderation messages are punitive rather than framed as quality control
- Automated content doesn't use production/factory framing (e.g., "Weekly Thread" instead of "Weekly Production Report")
- The sub's own automation contradicts the principles it advocates (e.g., closed-source bot configs while preaching open blueprints)

Do NOT flag:

- Simple automation that works but isn't fully themed yet (shipping beats perfection)
- Reddit platform limitations that prevent full factory theming
- Users interacting with bots in non-factory language (the theming applies to bot output, not user input)
- Automation that prioritizes function over theme when they conflict (a working QC system beats a beautifully themed broken one)
