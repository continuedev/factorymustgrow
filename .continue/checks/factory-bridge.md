---

r/factorymustgrow is primarily a software factory community — engineers building CI/CD pipelines, automated code review, deployment systems, and infrastructure as code. The Factorio connection is the shared mental model, not a 50/50 content split. Think ~80% software factory content, ~20% Factorio-inspired memes and analogies that resonate because the audience thinks in throughput.

The bridge standards:

1. **Software factories are the main line.** The primary content is real production alpha — configs, workflows, post-mortems, throughput reports from actual software systems. This is what the factory produces.
2. **Factorio memes hit different here.** Factorio content is welcome because this audience gets it on two levels. A meme about spaghetti belts lands harder when you've also debugged spaghetti microservices. But memes are seasoning, not the meal.
3. **The shared vocabulary is native.** Throughput, bottleneck, main bus, spaghetti, blueprint, megabase, production line — these terms are native to both worlds. Lean into the shared language.
4. **No gatekeeping either direction.** "Keep the game stuff out" and "this should be more about Factorio" are both violations. The ratio self-regulates through community value — the best software posts attract more discussion than memes.

Flag as failing if:

- Content balance tips heavily toward pure Factorio gameplay with no software connection
- Software factory content is treated as secondary or off-topic
- Factorio memes/references are dismissed or excluded entirely
- Content creates a hierarchy where one type of factory is more legitimate
- The sub drifts into generic software engineering with no factory framing

Do NOT flag:

- Pure Factorio memes that resonate with the builder audience (the 20% is valuable)
- Software posts that don't reference Factorio at all (most won't — that's the 80%)
- Leaning heavily on one side's terminology in a specific post
- Disagreements about whether a specific analogy holds (debate is healthy)
- Content from adjacent factory games (Satisfactory, Dyson Sphere Program, Shapez)
